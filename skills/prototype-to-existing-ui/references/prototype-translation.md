# 原型转换

当需要把产品原型、线框图、视觉稿、截图或 UI 图片转换成前端实现时，使用这份参考。

## 读取原型

识别这些信息：

```text
页面目的
主要用户操作
次要操作
信息层级
导航上下文
必需数据字段
交互控件
空、加载、错误、禁用和成功状态
响应式行为
```

如果原型保真度较低，从布局和文案中推断意图，再用项目已有 UI 语言表达。不要过度拟合粗略线框图中的间距、占位颜色或临时文案。

## 保留意图，适配风格

保留：

```text
工作流
内容层级
必需控件
页面关系
校验和状态要求
```

适配：

```text
颜色
圆角
间距
字体
图标风格
按钮变体
卡片和面板处理方式
表格密度
表单布局
```

项目风格应该让实现后的页面看起来像产品中原本就有的一部分。

## 实现规划

编辑前先决定：

1. 哪个已有路由、布局或模块应该承载这个页面。
2. 哪些已有组件可以覆盖原型中的元素。
3. 页面需要什么数据结构、mock 数据或 API 集成。
4. 当前必须实现哪些状态。
5. 项目期望哪些响应式断点或布局变化。

优先使用项目已有页面模式，而不是发明新的布局外壳。

## 响应式行为

遵循项目已有响应式约定。如果没有明显约定：

```text
桌面端：保留完整信息密度和并排控件
平板端：减少列数，保持主要操作可见
移动端：堆叠区块，折叠密集筛选，保持操作可触达
```

不要让文本重叠、控件尺寸不可预测地变化，也不要让原型里的固定宽度破坏小屏布局。

## 最终检查

报告完成前，确认：

```text
如果存在 .ui-reference/，实现已经使用它。
页面使用了已有 token、组件和样式约定。
原型的产品意图已经被表达。
结果避免了一次性视觉值，除非有明确理由。
常见状态已经处理，或已经明确说明未处理原因。
已经运行相关验证，或说明了无法运行的原因。
```
