# Agent 前端开发工作流程

> 本文档定义一个面向前端交付的 Agent 工作流程，涵盖从设计到代码审查的四个阶段。每个阶段明确输入、使用技能与输出，便于编排与复用。

---

## 阶段 1：前端设计

### 目标

基于产品需求与现有代码库，产出可指导后续开发的前端设计文档。

### 输入

| 输入项 | 说明 | 是否必需 |
|---|---|---|
| PRD（产品需求文档） | 描述业务目标与功能诉求 | 必需 |
| 原型图 / 视觉稿 | 页面布局、交互形态的参考 | 必需 |
| 时序图 | 关键流程的时序关系 | 可选 |
| 接口文档 | 后端提供的 API 契约 | 可选 |
| 代码 | 现有代码库，用于对齐风格与约束 | 必需 |

### 技能使用

- **codebase**：扫描与理解现有代码库结构、技术栈、组件库与约定。
- **caveman**：将需求与代码理解转化为结构化的设计思路与文档骨架。

### 输出

- **前端设计文档**，建议包含以下章节：
  - 概述与设计目标
  - 页面 / 组件拆分
  - 路由与状态设计
  - 数据流与接口对接方案
  - 复用组件与新增组件清单
  - 关键交互与异常处理
  - 与现有代码的兼容性说明

---

## 阶段 2：前端开发计划

### 目标

将设计文档拆解为可执行的开发计划，明确任务粒度、依赖与验收标准。

### 输入

| 输入项 | 说明 |
|---|---|
| 前端设计文档 | 阶段 1 的产出 |
| 代码 | 现有代码库，用于评估改动范围 |

### 技能使用

- **codebase**：评估代码现状，确定改动点与影响面。
- **caveman**：将设计内容结构化为有序任务与里程碑。

### 输出

- **前端开发计划文档**，建议包含以下章节：
  - 任务拆解（按页面 / 组件 / 模块）
  - 任务依赖关系与执行顺序
  - 预估工作量与里程碑
  - 每个任务的验收标准
  - 风险点与回滚策略

---

## 阶段 3：前端代码开发

### 目标

按开发计划本地迭代实现代码，保持与设计、计划一致。

### 输入

| 输入项 | 说明 |
|---|---|
| 前端开发计划文档 | 阶段 2 的产出，作为执行依据 |

### 技能使用

- **codebase**：在实现中持续参考与匹配现有代码风格与约定。
- **ponytail**：按计划逐项生成与迭代代码，保持代码可维护性与一致性。

### 输出

- **本地迭代代码**，要求：
  - 遵循现有技术栈与目录结构
  - 复用既有组件，避免重复实现
  - 通过本地 lint / typecheck / 测试
  - 与设计文档、开发计划保持一致

---

## 阶段 4：代码审查

### 目标

对产出代码进行 AI 代码审查，对照原始需求确认实现质量与风险。

### 输入

| 输入项 | 说明 |
|---|---|
| 代码 | 阶段 3 产出的本地代码变更 |
| 原始需求 | PRD 与原型图等，用于核对实现是否满足需求 |

### 技能使用

- **code-review-loop**：对代码变更执行循环式 AI 审查，输出分级问题与修复建议。

### 输出

- **审核文档**，建议包含以下内容：
  - 审查范围与变更概要
  - 分级问题清单（P0 / P1 / P2 等）
  - 每个问题的定位、原因与修复建议
  - 与原始需求的符合性结论
  - 通过 / 需修复后重审的结论

---

## 流程总览

```
阶段 1 前端设计           阶段 2 前端开发计划        阶段 3 前端代码开发        阶段 4 代码审查
┌───────────────┐       ┌───────────────┐        ┌───────────────┐        ┌───────────────┐
│ PRD / 原型图   │       │ 前端设计文档   │        │ 前端开发计划   │        │ 本地迭代代码   │
│ 时序图 / 接口  │──────▶│               │──────▶│               │──────▶│               │
│ 代码          │       │ 代码          │        │               │        │ 原始需求      │
└───────┬───────┘       └───────┬───────┘        └───────┬───────┘        └───────┬───────┘
        │                       │                        │                        │
   codebase / caveman       codebase / caveman      codebase / ponytail      code-review-loop
        │                       │                        │                        │
        ▼                       ▼                        ▼                        ▼
   前端设计文档            前端开发计划文档          本地迭代代码              审核文档
```

## 使用说明

1. 每个阶段按输入要求准备资料，缺失的可选项可不提供。
2. 阶段之间存在依赖关系，前序阶段输出为后序阶段输入，建议顺序执行。
3. 技能名称为约定标识，实际编排时按所在 Agent 平台的技能注册名调用。
4. 每阶段产出建议落盘为 Markdown 文档，便于追踪与归档。
