# General Skills Repository Design

## Goal

Turn this empty repository into a portable home for agent skills that can be reused by Codex, Claude Code, opencode, and other coding tools that understand Agent Skills-style folders.

## Architecture

The repository keeps one canonical skill source under `skills/`. Each skill is a folder with a required `SKILL.md` and optional `scripts/`, `references/`, and `assets/` resources. Tool-specific directories are generated by an install script rather than hand-maintained, so the same skill content can be copied into `.agents/skills`, `.claude/skills`, or `.opencode/skills`.

The repository also includes lightweight Node.js tooling for creating, validating, and installing skills. The tools use only built-in Node modules so the repo works without dependency installation.

## Components

- `skills/`: canonical skill source directories.
- `templates/skill/`: reusable files copied by the new-skill command.
- `scripts/new-skill.mjs`: creates a new skill folder from the template.
- `scripts/validate-skills.mjs`: validates skill folder names and `SKILL.md` frontmatter.
- `scripts/install-skills.mjs`: copies canonical skills into tool-specific target directories.
- `scripts/lib/skill-utils.mjs`: shared parsing, validation, path, and copy helpers.
- `tests/`: Node test coverage for the reusable script behavior.
- `docs/compatibility.md`: explains how each supported tool consumes the skills.
- `AGENTS.md`: durable instructions for future coding agents working in this repo.
- `README.md`: human-facing quick start.

## Data Flow

Authors create or edit skill folders in `skills/`. Validation reads each `SKILL.md`, checks its YAML frontmatter, and reports actionable errors. Installation reads the same canonical folders and mirrors them into the selected tool directories in a target project.

## Error Handling

CLI scripts fail with non-zero exit codes and short messages when skill names are invalid, required files are missing, frontmatter is malformed, or a target path cannot be written. Existing generated target directories are replaced per skill to avoid stale files, while unrelated directories in the target project are left untouched.

## Testing

Tests use Node's built-in `node:test` runner. Coverage focuses on skill name normalization, frontmatter parsing, validation failures, skill creation, and installation path mapping. Manual verification consists of creating a sample skill, validating the repository, and installing it into temporary tool directories.
